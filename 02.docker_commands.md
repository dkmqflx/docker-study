# Docker 명령어 완전 정복

---

## 전체 명령어 분류

| 분류 | 설명 |
|---|---|
| **General** | Docker 자체 정보 확인 |
| **Image Management** | 이미지 생성·조회·삭제 |
| **Container Management** | 컨테이너 실행·중지·삭제 |
| **Container Interaction** | 실행 중인 컨테이너와 상호작용 |
| **Debugging** | 문제 원인 파악 및 디버깅 |

## 1. General Docker Commands

Docker 자체의 상태와 정보를 확인하는 명령어입니다.

```bash
# 설치된 Docker의 버전을 출력합니다.
# Docker가 제대로 설치되었는지 확인할 때 가장 먼저 쓰는 명령어입니다.
docker --version

# Docker 시스템 전반의 정보를 출력합니다.
# 실행 중인 컨테이너 수, 이미지 수, 스토리지 드라이버, OS 정보 등을 확인할 수 있습니다.
docker info

# 사용 가능한 모든 Docker 명령어와 옵션 목록을 출력합니다.
# 명령어가 생각나지 않을 때 참고하세요.
docker help

# 특정 명령어의 상세 사용법을 확인합니다.
# 예: docker help run → run 명령어의 모든 옵션을 볼 수 있습니다.
docker help [command]
```

## 2. Image Management

이미지를 다운로드하고, 빌드하고, 관리하는 명령어입니다.

> **복습:** Image는 컨테이너를 만들기 위한 읽기 전용 틀(붕어빵 틀)입니다.

```bash
# 로컬 머신에 저장된 모든 이미지 목록을 출력합니다.
# IMAGE ID, 이름, 태그, 크기, 생성 시각을 보여줍니다.
docker images

# Docker Hub(또는 다른 레지스트리)에서 이미지를 다운로드합니다.
# 태그를 생략하면 자동으로 최신 버전(latest)을 받습니다.
docker pull [image-name]
docker pull nginx          # 예: Nginx 최신 버전 다운로드
docker pull nginx:1.25     # 예: Nginx 1.25 특정 버전 다운로드

# 현재 디렉토리의 Dockerfile을 사용해 이미지를 빌드합니다.
# -t 옵션으로 이미지에 이름(태그)을 지정합니다.
# 마지막의 '.'은 Dockerfile이 있는 경로(현재 디렉토리)를 의미합니다.
docker build -t [image-name] .
docker build -t my-app .         # 예: my-app이라는 이름으로 빌드
docker build -t my-app:1.0 .     # 예: 버전 태그(1.0)까지 지정해서 빌드

# 이미지에 새로운 이름이나 버전 태그를 붙입니다.
# 같은 이미지를 Docker Hub에 올리기 전에 이름을 정리할 때 자주 사용합니다.
docker tag [source-image] [target-image:tag]
docker tag my-app myusername/my-app:1.0  # 예: Docker Hub 업로드 전 태그 지정

# 이미지를 Docker Hub(또는 다른 레지스트리)에 업로드합니다.
# 업로드 전에 docker login으로 로그인이 되어 있어야 합니다.
docker push [image-name]
docker push myusername/my-app:1.0

# 이미지를 삭제합니다. ID 또는 이름으로 지정할 수 있습니다.
# 해당 이미지로 만든 컨테이너가 실행 중이면 삭제할 수 없습니다.
docker rmi [image-id or image-name]
docker rmi nginx           # 예: nginx 이미지 삭제
docker rmi abc123def456    # 예: IMAGE ID로 삭제

# 사용하지 않는 이미지를 한번에 모두 삭제합니다. (디스크 정리 시 유용)
docker image prune
docker image prune -a      # 컨테이너가 없는 모든 이미지 삭제
```

## 3. Container Management

컨테이너를 실행하고, 중지하고, 삭제하는 명령어입니다.

> **복습:** Container는 Image로부터 만들어진, 실제로 실행 중인 앱 인스턴스(붕어빵)입니다.

```bash
# 이미지로부터 새 컨테이너를 만들고 실행합니다.
# 가장 자주 쓰는 명령어입니다.
docker run [image-name]

# 주요 옵션:
#   -d       : 백그라운드에서 실행 (detached mode) — 터미널이 점유되지 않음
#   -p       : 포트 연결 [내PC포트]:[컨테이너포트]
#   --name   : 컨테이너에 이름 지정
#   -e       : 환경변수 설정
#   -v       : 볼륨 마운트 (로컬 폴더 ↔ 컨테이너 폴더 연결)
#   --rm     : 컨테이너 종료 시 자동 삭제
#   -it      : 인터랙티브 터미널 모드 (컨테이너 내부에 직접 접속할 때)

docker run -d -p 8080:80 nginx
# 예: nginx를 백그라운드로 실행, 내 PC의 8080 포트 → 컨테이너 80 포트 연결

docker run -d -p 3000:3000 --name my-app my-app:1.0
# 예: my-app 이미지를 'my-app'이라는 이름으로 실행

docker run -d -e MYSQL_ROOT_PASSWORD=secret -p 3306:3306 mysql
# 예: MySQL을 실행하면서 환경변수로 비밀번호 설정

# 실행 중인 컨테이너 목록을 출력합니다.
docker ps

# 중지된 컨테이너를 포함한 모든 컨테이너 목록을 출력합니다.
docker ps -a

# 실행 중인 컨테이너를 중지합니다. (컨테이너 데이터는 유지됨)
docker stop [container-id or name]
docker stop my-app

# 중지된 컨테이너를 다시 시작합니다.
docker start [container-id or name]
docker start my-app

# 실행 중인 컨테이너를 즉시 강제 종료합니다.
# docker stop이 응답이 없을 때 사용합니다.
docker kill [container-id or name]

# 중지된 컨테이너를 삭제합니다.
# 실행 중인 컨테이너는 먼저 stop한 뒤 삭제해야 합니다.
docker rm [container-id or name]
docker rm my-app

# 실행 중인 컨테이너를 강제로 삭제합니다. (-f: force)
docker rm -f [container-id or name]

# 중지된 모든 컨테이너를 한번에 삭제합니다. (디스크 정리 시 유용)
docker container prune
```

## 4. Container Interaction

이미 실행 중인 컨테이너 안으로 들어가거나, 파일을 주고받는 명령어입니다.

```bash
# 실행 중인 컨테이너 안에서 명령어를 실행합니다.
docker exec [container-id or name] [command]

docker exec my-app ls /app
# 예: my-app 컨테이너 안의 /app 폴더 내용 확인

# -it 옵션으로 컨테이너 내부에 쉘로 직접 접속합니다.
# 마치 SSH로 서버에 접속하는 것과 같습니다.
docker exec -it my-app bash
# 예: bash 쉘로 접속 (나올 때는 exit 또는 Ctrl+D)

docker exec -it my-app sh
# 예: bash가 없는 Alpine 기반 이미지는 sh 사용

# 로컬 파일을 컨테이너 안으로 복사하거나, 컨테이너 파일을 로컬로 꺼냅니다.
docker cp [local-path] [container-name]:[container-path]   # 로컬 → 컨테이너
docker cp [container-name]:[container-path] [local-path]   # 컨테이너 → 로컬

docker cp ./config.json my-app:/app/config.json
# 예: 로컬의 config.json을 컨테이너의 /app 폴더로 복사

docker cp my-app:/app/logs ./logs
# 예: 컨테이너의 /app/logs 폴더를 로컬로 꺼내기
```

## 5. Debugging Commands

컨테이너에 문제가 생겼을 때 원인을 파악하는 명령어입니다.

```bash
# 컨테이너의 로그(표준 출력)를 출력합니다.
# 앱이 왜 안 되는지 확인할 때 가장 먼저 실행하는 명령어입니다.
docker logs [container-id or name]
docker logs my-app

# -f 옵션으로 실시간으로 로그를 스트리밍합니다. (tail -f 와 동일)
docker logs -f my-app

# 마지막 N줄만 출력합니다.
docker logs --tail 100 my-app

# 컨테이너의 상세 정보를 JSON 형식으로 출력합니다.
# IP 주소, 환경변수, 볼륨, 네트워크 설정 등을 확인할 수 있습니다.
docker inspect [container-id or name]
docker inspect my-app

# 특정 정보만 추출할 때 --format 옵션을 사용합니다.
docker inspect --format='{{.NetworkSettings.IPAddress}}' my-app
# 예: 컨테이너의 IP 주소만 출력

# 실행 중인 컨테이너의 CPU, 메모리, 네트워크 사용량을 실시간으로 모니터링합니다.
# 리눅스의 htop과 비슷한 역할입니다.
docker stats
docker stats my-app   # 특정 컨테이너만 모니터링

# 컨테이너 안에서 실행 중인 프로세스 목록을 출력합니다.
docker top [container-id or name]
docker top my-app

# 이미지가 어떤 레이어(Layer)로 구성되어 있는지 이력을 출력합니다.
# Dockerfile의 각 명령어가 어떤 레이어를 만들었는지 확인할 수 있습니다.
docker history [image-name]
docker history my-app
```

## 명령어 전체 요약

```bash
# ── General ──────────────────────────────────────────────────
docker --version              # Docker 버전 확인
docker info                   # 시스템 전체 정보
docker help                   # 전체 명령어 도움말

# ── Image ────────────────────────────────────────────────────
docker images                 # 이미지 목록
docker pull [image]           # 이미지 다운로드
docker build -t [name] .      # Dockerfile로 이미지 빌드
docker tag [src] [dst:tag]    # 이미지 태그 변경
docker push [image]           # 이미지 업로드
docker rmi [image]            # 이미지 삭제
docker image prune -a         # 미사용 이미지 일괄 삭제

# ── Container ────────────────────────────────────────────────
docker run -d -p [h]:[c] [image]   # 컨테이너 실행
docker ps                     # 실행 중인 컨테이너 목록
docker ps -a                  # 전체 컨테이너 목록 (중지 포함)
docker stop [container]       # 컨테이너 중지
docker start [container]      # 컨테이너 재시작
docker kill [container]       # 컨테이너 강제 종료
docker rm [container]         # 컨테이너 삭제
docker container prune        # 중지된 컨테이너 일괄 삭제

# ── Interaction ──────────────────────────────────────────────
docker exec -it [container] bash   # 컨테이너 내부 접속
docker exec [container] [cmd]      # 컨테이너 안에서 명령 실행
docker cp [src] [dst]              # 파일 복사 (로컬 ↔ 컨테이너)

# ── Debugging ────────────────────────────────────────────────
docker logs -f [container]    # 로그 실시간 확인
docker inspect [container]    # 컨테이너 상세 정보 (JSON)
docker stats [container]      # 리소스 사용량 모니터링
docker top [container]        # 실행 중인 프로세스 확인
docker history [image]        # 이미지 레이어 이력
```

## 자주 쓰는 패턴

### 컨테이너 실행 후 바로 내부 접속
```bash
docker run -it ubuntu bash
# Ubuntu 컨테이너를 만들고 bash 쉘로 바로 접속
# 종료(exit)하면 컨테이너도 자동으로 중지됨
```

### 테스트 후 자동 삭제
```bash
docker run --rm -p 8080:80 nginx
# --rm: 컨테이너 종료 시 자동 삭제 → 테스트할 때 컨테이너가 쌓이지 않음
```

### 문제 발생 시 빠른 확인 루틴
```bash
docker ps -a               # 1. 컨테이너 상태 확인 (중지됐는지 등)
docker logs my-app         # 2. 로그에서 에러 메시지 확인
docker inspect my-app      # 3. 설정 값(포트, 환경변수) 확인
docker exec -it my-app sh  # 4. 직접 접속해서 내부 상태 확인
```
